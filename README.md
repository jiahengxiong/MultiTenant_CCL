# MultiTenant

This repository is a research-oriented framework for multi-tenant collective communication in a leaf-spine datacenter.

The codebase is organized around one baseline and one proposed method family:

- `Harmonics Baseline`: the baseline scheduler.
- `Proposed Mapping ILP`: the exact formulation used for small-scale validation.
- `Proposed Mapping CG`: the scalable formulation used for larger studies.

## Layout

- `multitenant/`
  The normalized Python package for topology construction, workload generation, simulation adapters, baseline methods, proposed mapping solvers, plotting helpers, and diagnostics.

- `scripts/`
  The only intended command-line entry points for experiments, plotting, debugging, and topology inspection.

- `CCL_Simulator/`
  The packet-level event-driven simulator core that executes the communication policies generated by this project.

## Main Scripts

- `python scripts/run_baseline_vs_proposed_mapping.py`
  Runs the main comparison between baseline methods and the proposed mapping CG method.

- `python scripts/run_proposed_mapping_small_scale.py`
  Runs the small-scale validation where the exact proposed mapping ILP is used as the reference.

- `python scripts/run_proposed_mapping_large_scale.py`
  Runs the large-scale study for the scalable proposed mapping CG method.

- `python scripts/plot_baseline_vs_proposed_mapping.py`
  Plots the baseline-vs-proposed comparison figures from the saved JSON results.

## Debug Scripts

- `python scripts/debug_harmonics_baseline.py`
- `python scripts/debug_proposed_mapping_ilp.py`
- `python scripts/inspect_topology.py`

## Output Convention

Experiment scripts write generated artifacts under `results/`, which is ignored by git so that the repository stays source-only and clean.
